## 1. Requirements (Context)
- **Goal**: 评估当前项目是否可以改造成“在游戏 Mod 设置页配置指令识别”，并支持中英双语。
- **In-Scope**:
  - 识别当前命令词定义位置
  - 识别当前设置持久化方式
  - 识别 HK Mod 菜单与本地化可接入点
  - 评估中英双语识别对当前 Sherpa KWS 路线的影响
- **Out-of-Scope**:
  - 本轮不直接改代码
  - 本轮不落地 UI 实现
  - 本轮不训练或替换模型

## 1.5 Code Map (Project Topology)
- **Core Logic**:
  - `HkVoiceMod/Recognition/Sherpa/SherpaKeywordSpotterBackend.cs`: Sherpa 关键词检测后端；当前内置 `CommandLookup`，把识别出的 keyword 映射到 `VoiceCommand`。
  - `HkVoiceMod/Commands/VoiceCommandMap.cs`: `VoiceCommand -> KeyActionProfile` 动作映射。
  - `HkVoiceMod/Runtime/VoiceRuntimeController.cs`: 主线程消费识别结果并驱动输入注入。
  - `HkVoiceMod/Input/HeroActionInputInjector.cs`: 把动作映射真正写入 `HeroActions` / `moveVector`。
- **Entry Points**:
  - `HkVoiceMod/HkVoiceMod.cs`: Mod 入口，当前实现了 `IGlobalSettings<VoiceModSettings>`，说明已具备全局配置持久化基础。
- **Data Models**:
  - `HkVoiceMod/VoiceModSettings.cs`: 当前只保存开关、模型路径、时长、日志等，没有命令别名/本地化设置。
  - `HkVoiceMod/Commands/VoiceCommand.cs`: 固定 11 个命令枚举。
- **Assets / Recognition Data**:
  - `HkVoiceMod/assets/sherpa-kws-cn/keywords_raw.txt`: 原始中文关键词表。
  - `HkVoiceMod/assets/sherpa-kws-cn/keywords.txt`: 按 `ppinyin` 编码后的运行时关键词表。
- **Dependencies**:
  - `Modding API`: 当前已使用 `Mod` 与 `IGlobalSettings<T>`；可继续接入 `IMenuMod` / `ICustomMenuMod`。
  - `SherpaOnnx.KeywordSpotter`: 当前项目使用中文 KWS 模型，关键词文件由 `keywords.txt` 驱动。
  - `NAudio`: 麦克风采集。

## 2. Architecture (Optional - Populated in INNOVATE)
- **当前事实**:
  - 现状不是“可配置指令系统”，而是“固定中文关键词 + 固定代码查表”。
  - 命令识别白名单有两层：
    1. `keywords.txt` 决定模型允许触发哪些词；
    2. `CommandLookup` 决定这些词映射到哪个 `VoiceCommand`。
  - 仅改 `VoiceModSettings` 还不够，必须同步处理关键词文件与内存查表。
- **菜单能力评估**:
  - HK Mod API 的基础 `IMenuMod.GetMenuData(...)` 只提供 `Name / Description / Values / Saver / Loader` 这种枚举式菜单项，适合开关、下拉、枚举，不适合自由文本输入。
  - 若只是“每个动作在几组中文/英文预设词之间切换”，基础 Mod 菜单可做。
  - 若要求“玩家直接在游戏里输入任意自定义口令”，更适合 `ICustomMenuMod` 或额外依赖 `MenuChanger` 一类更强菜单系统。
- **双语能力评估**:
  - 菜单文案双语：可行。可通过 `LanguageGetHook` 或菜单 API 文案层实现。
  - 识别词双语：**当前中文模型不能无成本直接扩成稳健中英双语**。
    - 当前仓库使用的是 `sherpa-kws-cn`，`keywords.txt` 由 `ppinyin` 生成，明显是中文路线。
    - Sherpa 官方文档显示中英混合 KWS 需要换到 `zh-en` 预训练模型，并使用 `phone+ppinyin` + 英文 lexicon 的关键词转换流程。
  - 结论：UI 双语容易，识别双语需要更换模型/关键词生成链路。
- **推荐策略**:
  - Phase A：先做“菜单页配置 + 中文自定义/预设词”。
  - Phase B：切换到 Sherpa 官方 `zh-en` KWS 模型后，再做真正的中英混合识别。

## Interim Verdict
- **能改，但要分清“菜单双语”和“识别双语”是两件事。**
- **在当前架构下：**
  - 做“Mod 设置页配置预设指令/别名”= 可行，改动中等；
  - 做“设置页自由输入任意指令”= 基础菜单不够，需自定义菜单；
  - 做“真正中英双语识别”= 当前中文模型不够，需要换 `zh-en` KWS 模型及关键词生成链路。
