# SDD Spec: 录制页按键序列可见性修复

## 0. 🚨 Open Questions (MUST BE CLEAR BEFORE CODING)
- None

## 1. Requirements (Context)
- **Goal**: 修复录制页“当前完整事件序列”中动作内容不显示的问题，确保用户能看到每条按下/松开事件对应的具体动作文本。
- **In-Scope**:
  - 仅修复录制 modal 中事件行的动作值文本生成逻辑。
  - 恢复“打开录制页时隐藏底层设置页”的既有行为。
  - 保持现有列表布局、按钮 ornament、主题、间隔行、其它页面摘要与提示文案不变。
  - 保证文本来源为稳定、可渲染的纯文本动作名。
- **Out-of-Scope**:
  - 不重构整套主题系统。
  - 不修改录制状态机、事件模型、按钮交互。
  - 不改动录制页之外的显示策略。

## 1.5 Code Map (Project Topology)
- **Core Logic**:
  - `HkVoiceMod/UI/VoiceSettingsWindowController.cs`: 录制页 UI 构建、事件列表渲染、录制行值文本生成。
  - `HkVoiceMod/Menu/GameKeybindNameResolver.cs`: 动作显示名与 `PlayerAction` 查询入口。
- **Entry Points**:
  - `BuildRecordingModal(...)`: 创建录制 modal 与“当前完整事件序列”列表。
  - `RefreshRecordingStepRows()`: 每帧刷新事件行文本。
  - `BuildRecordingStepValueText(...)`: 录制行值文本格式化入口。
- **Data Models**:
  - `VoiceMacroConfig.KeyEvents`: 录制事件源数据。
  - `VoiceMacroKeyEvent`: 单条动作事件，包含 `ActionButton` / `EventKind` / `DelayBeforeMilliseconds` / `PairId`。
  - `RecordingTimelineRowModel`: UI 投影行模型，区分 delay 行与 action 行。
- **Dependencies**:
  - `InputHandler.ActionButtonToPlayerAction(...)`: 从动作枚举取回游戏 `PlayerAction`。
  - `UnityEngine.UI.Text`: 当前录制行文本渲染组件。

## 2. Architecture (Optional - Populated in INNOVATE)

## 4. Execution Notes
- 最新根因结论：录制行的 value 区此前错误走了 `CreateButton(... useSettingsPageStyle: true)` 分支；该分支会把按钮底图清空并挂 `SettingsPageButtonStyle`，更适合普通设置页按钮，不适合录制行这种左对齐内容格子。
- 因为录制 modal / preview / row 本身还带透明主题，叠加设置页未隐藏时，会进一步放大 value 文本不可见问题；因此本轮同时恢复了打开录制页时隐藏设置页主体的既有逻辑。
- 实际落地：
  - 恢复 `_windowCanvasGroup + SetWindowPageVisible(...)`，在录制 modal 打开/关闭时隐藏与恢复设置页主体。
  - 没有保留“普通按钮路径”方案，因为它会把整张按钮 sprite 拉伸进 value 区并丢失原有 hover/ornament 效果。
  - 也没有保留“双 Text overlay”方案；对比排查确认它让 `RecordingStepValue-*` 成为页面里唯一一个拥有两个 `Text` 子节点的 styled button，和正常按钮结构不一致。
- 当前最新实现改为：`RecordingStepValue-*` 仍走 `useSettingsPageStyle: true`，并恢复为与其它正常 styled button 相同的单 `Text` 结构；同时移除录制行特有的 overlay 路径，不再让它脱离 `SettingsPageButtonStyle.LabelText` 的主题链路。
- 保留录制页 action 行的局部纯文本动作名 helper，避免共享显示名里潜在 glyph 风险。
- 由于最新实机截图仍显示“只有左右 ornament、中央无文字”，当前已停止继续猜测，并新增一次性运行时探针：在录制页首次出现事件行时，将录制行按钮与“开始录制”按钮的 `Button/Text` 运行时状态（text、color、rect、child tree、style label 绑定）一起写入 `ModLog`。
