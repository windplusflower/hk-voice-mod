# SDD Spec: 录制界面方向键中文映射

## 0. 🚨 Open Questions (MUST BE CLEAR BEFORE CODING)
- None

## 1. Requirements (Context)
- **Goal**: 让录制界面中 `UP/DOWN/LEFT/RIGHT` 的显示文案与游戏键位设置页保持一致，直接读取游戏当前键位页上的中文动作名，避免显示 `Up/Down/Left/Right` 英文文本。
- **In-Scope**:
  - 修复录制弹窗中方向动作的展示来源。
  - 保持录制逻辑、动作解析逻辑、宏数据结构不变，只调整显示名解析链路。
  - 方向键显示需与游戏键位设置页中的中文文案一致，即 `上/下/左/右`，而不是 mod 自定义的 `上移/下移/左移/右移` 或英文 `Up/Down/Left/Right`。
- **Out-of-Scope**:
  - 不修改录制状态机。
  - 不修改宏回放注入逻辑。
  - 不扩展新的可录制动作集合。

## 1.5 Code Map (Project Topology)
- **Core Logic**:
  - `HkVoiceMod/Menu/GameKeybindNameResolver.cs`: 录制页动作显示名解析器。`GetDisplayName(...)` 先尝试方向键专用的键位页标签抓取，再回退到 `InputHandler.ActionButtonLocalizedKey(...)` / `PlayerAction.Name` / 本地默认文案。
  - `HkVoiceMod/Menu/VoiceMacroCaptureService.cs`: 录制服务，持有 `GameKeybindNameResolver` 实例，并在录制/编辑时复用它完成按键解析和显示。
  - `HkVoiceMod/UI/VoiceSettingsWindowController.cs`: 录制弹窗渲染入口，`BuildRecordingActionValueText(...)` 直接调用 `resolver.GetDisplayName(actionButton)` 把动作名显示到预览列表。
- **Entry Points**:
  - `HkVoiceMod/UI/VoiceSettingsWindowController.cs:778`: `BuildRecordingActionValueText(GlobalEnums.HeroActionButton actionButton, GameKeybindNameResolver resolver)`。
  - `HkVoiceMod/Menu/GameKeybindNameResolver.cs:36`: `GetDisplayName(GlobalEnums.HeroActionButton actionButton)`。
  - `HkVoiceMod/Menu/GameKeybindNameResolver.cs:102`: `TryGetKeyboardMenuLabel(GlobalEnums.HeroActionButton actionButton, out string displayName)`。
  - `HkVoiceMod/Menu/GameKeybindNameResolver.cs:115`: `RefreshKeyboardMenuLabelCache()`，扫描活动中的 `MappableKey` UI 并缓存标签文本。
- **Data Models**:
  - `GlobalEnums.HeroActionButton`: 录制步骤里保存的游戏动作枚举，方向键对应 `UP/DOWN/LEFT/RIGHT`。
  - `_keyboardMenuLabelCache`: 方向动作到键位页标签文案的运行时缓存。
- **Dependencies**:
  - `InputHandler.ActionButtonToPlayerAction(...)`: 动作枚举到 `PlayerAction` 的原生映射。
  - `InputHandler.ActionButtonLocalizedKey(...)`: 当前实现的通用显示名来源之一。
  - `Language.Language.Get(..., "MainMenu")`: 尝试把语言键翻译为当前语言文本。
  - `MappableKey` + `Text`: 游戏键位设置页 UI 组件，方向键中文标签实际来自这里附近的文本节点。

## 2. Architecture (Optional - Populated in INNOVATE)
- 当前已确认的现实行为：
  - 方向键是特殊分支，不应依赖 `ActionButtonLocalizedKey(...)` 产出最终显示名。
  - 游戏键位页中的 `上/下/左/右` 更像是 UI 标签文本，而不是稳定可翻译的统一动作名表项。
  - 现有实现已经尝试通过扫描活动中的 `MappableKey` 抓取这些标签，但在录制界面显示英文，说明该抓取链路在当前时机未命中，随后回退到了英文动作名。
- 当前根因线索：
  - `RefreshKeyboardMenuLabelCache()` 只有在场景中存在且激活的 `MappableKey` 时才会刷新缓存。
  - 一旦录制弹窗出现时键位设置页 UI 不再处于可扫描状态，方向键标签缓存无法建立或已过期。
  - 回退链路随后使用 `localizedKey -> translatedName -> action.Name`，对方向键会落到英文 `Up/Down/Left/Right`。

## 4. Current Findings Snapshot
- 录制预览值由 `VoiceSettingsWindowController.BuildRecordingActionValueText(...)` 直接读取 `GameKeybindNameResolver.GetDisplayName(...)`。
- `GameKeybindNameResolver` 已对方向键做“从键位设置菜单抓标签”的特殊处理，说明仓库当前设计目标本来就是读游戏键位页文本，而不是写死方向文案。
- 问题不在录制事件内容；方向动作枚举仍是正确的 `UP/DOWN/LEFT/RIGHT`。
- 问题集中在显示名解析时机与回退策略：当键位页标签抓取失败时，当前实现会退回英文动作名。

## 5. Verified Conclusion
- 这是一个**展示层缺陷**，不是录制/回放功能缺陷。
- 需要在后续 PLAN 中明确：方向键显示名必须优先取自游戏键位页缓存，并在缓存缺失时使用与键位页一致的中文兜底，而不能继续退回英文动作名。

## 6. Planned Change Summary
- 改动范围限定为 `HkVoiceMod/Menu/GameKeybindNameResolver.cs` 的显示名解析分支。
- 修复原则是“优先读取游戏键位页真实标签，读取不到时保持与该页一致的中文结果”，而不是把方向键继续暴露为英文动作名。
- 这样可以同时覆盖两种运行时：
  - 键位页 UI 当前可见：直接吃到游戏真实标签文本。
  - 键位页 UI 当前不可见或未预热成功：仍返回与键位页一致的中文方向名，避免展示退化。
