## 2. 推荐实现边界
- 目标不是做 OS 键盘模拟，而是直接驱动 `InputHandler.Instance?.inputActions` / `HeroActions`
- 识别路线按既定结论执行：
  - 验证版：`Vosk + 中文命令白名单`
  - 正式版预留：`DS-CNN / Tiny CNN` 替换识别后端
- 线程模型固定：
  - 子线程：录音 + 识别
  - 主线程：消费识别结果 + 注入 HeroActions

## 4. 目标架构总图
```text
Vosk 麦克风线程
  -> Vosk 识别

  -> HeroActions(up/down/left/right/attack/jump/dash/cast)
```

## 5. 关键模块索引

### 5.1 Entry / 生命周期
- `HkVoiceMod/HkVoiceMod.cs`
  - Mod 入口
  - 初始化运行时对象
  - 管理 settings / unload

### 5.2 识别层
- `HkVoiceMod/Recognition/IVoiceRecognitionBackend.cs`
  - 识别后端抽象，保证后续可从 Vosk 替换为 DS-CNN
- `HkVoiceMod/Recognition/Vosk/VoskVoiceRecognitionBackend.cs`
  - Vosk 验证版实现
  - 负责麦克风、模型、结果解析、输出队列

### 5.3 指令层
- `HkVoiceMod/Commands/VoiceCommand.cs`
  - 10 个动作词 + 1 个控制词 `停`
- `HkVoiceMod/Commands/KeyActionProfile.cs`
  - 命令到按键行为的查表描述
- `HkVoiceMod/Commands/VoiceCommandMap.cs`
  - 固定映射表

### 5.4 输入注入层
- `HkVoiceMod/Input/HeroActionInputInjector.cs`
  - 统一处理短按、0.5 秒定时长按、左右持续按住、`停` 释放
  - 同步 `HeroActions` 单键状态
  - 同步 `moveVector`，避免 `HeroController` 读取方向轴时失配

### 5.5 主线程协调层
- `HkVoiceMod/Runtime/VoiceRuntimeController.cs`
  - 消费队列
  - 调用映射与注入器
  - 控制后台识别后端生命周期

## 6. 外部依赖
- Hollow Knight Modding API
- `InputHandler` / `HeroActions` / `InControl`
- Vosk C# 绑定
- 麦克风采集库（验证版优先 `NAudio`，如执行期发现平台约束再回 Spec 修订）

## 7. 主要技术风险
- `HeroController` 同时读取 `inputActions.left/right/up/down` 与 `moveVector.Vector`，注入时必须两边同步
- Vosk 中文模型体积远大于“几十 KB”，因此验证版必须与正式版目标分离
- Unity / HK 主线程对象不能被识别线程直接访问
- `停` 只应取消持续按键，不应误杀正在执行的 0.5 秒定时动作
